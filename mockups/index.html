Design Decisions & Assumptions

Assumptions Made (PRD gaps filled)

●Slippage visual distinction: PRD says "visually distinct from clinical flags" — implemented using an orange/amber gradient header (vs teal for clinical), a different card accent color, and explicit "Engagement risk, not clinical emergency" explainer text.
●Food log screen layout: PRD describes "dish name, estimated macros, carb flag, brief note" — extended to include a day-selector tab bar (last 5 days), daily totals vs protocol, and per-meal cards since this is more useful in practice.
●Coach notes as full screen: PRD says "modal or side panel" — mocked as a full mobile screen since modals on small screens are poor UX. The design supports adaptation to a modal on tablet/desktop.
●Operator Dashboard is desktop-first: PRD is unclear on device — operators are assumed to use desktop (data-heavy analytics view). Mobile could be a separate P2 screen.
●Bottom nav order: Brief / Patients / Alerts / Notes / Program — ordered by daily-use frequency for coaches. Program nav leads to operator dashboard (role-gated in real implementation).
●Login role selector: PRD lists Coach, Operator, Doctor roles. Added role selector on login screen since this shapes the landing screen (Coach → Morning Brief, Operator → Dashboard, Doctor → P2 read-only view).
●Logging frequency visualization: Shown as a 30-day dot grid (like GitHub contribution graph) — 3 states: logged (teal), partial (light teal), missed (gray). Intuitive at a glance without requiring explanation.
●Glucose chart — CSS-only bar chart: Uses stacked divs with color coding (teal/amber/red) since no external chart library is used. Heights represent proportional FBS values.
